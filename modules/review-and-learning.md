# 模块：审计与经验沉淀

## 目的

审计阶段产物，并判断是通过、细化、回退，还是形成经验沉淀建议。

## 何时激活

- PRD、方案、原型设计 Brief 或模块产物已经完成。
- 用户要求审计。
- 流程中暴露了重复错误或有价值经验。

## 执行步骤

1. 选择相关 Rubric。
2. 优先检查方向级问题。
3. 检查产品逻辑和 PRD 完整性。
4. 判断通过、细化或回退。
5. 如有价值，提出经验沉淀建议。

## 输出

- 按严重程度排序的问题
- 必须返工项
- 必要时的回退建议
- 通过 / 细化 / 回退结论
- 可选经验沉淀建议

长期经验沉淀必须经过用户确认。
