# 模块：产品现状理解

## 目的

在做新设计前，先理解产品当前是怎么工作的。

## 何时激活

- 方案依赖已有产品行为。
- 现有文档、截图、产品界面或历史决策可能影响设计。
- 需求是把已有能力扩展到新对象、新角色、新入口或新流程。

## 输入

- 产品文档
- 截图或用户流程
- 历史决策
- 帮助中心或支持材料
- 已知约束

## 执行步骤

1. 识别当前对象、角色、入口和流程。
2. 记录当前生命周期和状态迁移。
3. 记录当前权限和可见性规则。
4. 识别当前用户侧表达。
5. 区分已验证事实和假设。
6. 标记不同来源之间的冲突。

## 输出

- 当前行为摘要
- 当前约束
- 相关对象模型
- 生命周期或状态事实
- 来源冲突
- 未知项
- 对方案设计的影响
