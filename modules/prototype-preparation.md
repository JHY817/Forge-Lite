# 模块：原型设计准备

## 目的

在方案方向确认后，为设计或原型工具准备结构化输入。

## 何时激活

- 需要验证界面入口、布局、信息层级、交互状态或用户流程。
- PRD 完成前需要目标图、线框图或可交互原型。

## 边界

这个模块只准备输入，不替代已配置的设计/原型工具、设计系统或视觉 QA。FORGE Lite 不绑定具体工具供应商。

## 执行步骤

1. 识别原型验证场景。
2. 梳理关键用户路径。
3. 定义页面或流程状态。
4. 定义状态之间的跳转关系。
5. 收集设计约束。
6. 生成原型设计 Brief。

## 输出

- 场景地图
- 交互状态矩阵
- 页面或流程索引
- 原型设计 Brief
- 用户确认点
- 待确认设计问题

## 输出承接

设计或原型工具生成目标图、线框图、可交互原型或 QA 结果后，Agent 必须先执行 `rubrics/prototype-output-quality.md`：

- 检查方案保真、场景覆盖和状态闭环；
- 检查是否擅自新增功能或改变动作语义；
- 检查是否符合使用者配置的设计约束；
- 方向问题回退方案阶段，视觉问题返回原型工具返工；
- Agent 验收通过后再进入用户确认。

用户确认不能替代 Agent QA。
