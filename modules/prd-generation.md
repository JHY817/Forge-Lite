# 模块：PRD 生成

## 目的

把已确认的产品决策转成可交付、可实现、可测试的产品需求文档。

## 何时激活

- 方案方向已经确认。
- 关键产品逻辑和范围已经清楚。
- 剩余未知项可以被明确列出。

## 执行步骤

1. 复述背景和目标。
2. 定义范围和非目标。
3. 描述用户和场景。
4. 编写功能需求。
5. 编写产品逻辑、状态和边界条件。
6. 补充验收标准。
7. 列出依赖、风险和待确认问题。

## 输出

使用：

```text
templates/prd.md
```

PRD 必须区分：

- 已确认事实
- 产品决策
- 假设
- 待确认问题
- 不做范围
