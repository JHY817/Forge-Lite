# 单模块任务流程

当用户只要求完成一个聚焦任务时，使用这个流程。

例如：

- 分析产品现状。
- 做竞品分析。
- 基于代码做逆向工程。
- 审一份 PRD。
- 判断是否需要补数据。

## 目标

完成聚焦任务，不把所有请求都扩展成完整 PRD 流程。

## 流程

```text
1. 明确目标模块
2. 读取当前状态并装配必要证据
3. 判断是否需要方向门禁
4. 检查输入是否足够
5. 执行选定模块和对应 Rubric
6. 记录状态迁移与交接摘要
7. 按 auto / notify / approve 判断下一步
```

## 开始前检查

执行前先回答：

- 用户真正请求的是哪个模块？
- 目标是否足够清楚？
- 是否触碰对象模型、生命周期、权限、状态机或核心路径？
- 是否需要先运行其他模块？
- 哪些来源可以作为事实依据？

## 输出要求

最终输出应包括：

- 结论
- 证据或来源边界
- 未知项
- 风险
- 建议下一步
- 当前状态迁移和失效产物

不要自行把单模块任务扩展成完整 PRD。扩展范围需要 approve；范围内低风险补证和修复不重复询问用户。

逆向工程结论进入产品判断前使用 `rubrics/reverse-engineering-quality.md`；原型输出使用 `rubrics/prototype-output-quality.md`。
