# 审计与复盘流程

当需要审计 PRD、方案、原型设计 Brief 或阶段产物时，使用这个流程。

## 目标

识别产品风险、缺失证据、方向问题，以及是否需要返工或回退。

## 流程

```text
1. 判断产物类型
2. 选择相关 Rubric
3. 按严重程度审计
4. 判断通过 / 细化 / 回退
5. 提出经验沉淀建议
```

## 审计优先级

先讲问题，不要先写表扬。

优先检查：

1. 方向级问题。
2. 缺失或无法验证的事实。
3. 对象模型、生命周期、权限、状态机问题。
4. 依赖系统隐式补偿的规则。
5. 实现边界不清的问题。
6. 缺失验收标准的问题。
7. 表达和结构问题。

## 输出格式

- 按严重程度排序的问题。
- 证据或判断依据。
- 建议动作。
- 待确认问题。
- 通过 / 细化 / 回退建议。
- 可选经验沉淀建议。

长期经验沉淀必须经过用户确认。
