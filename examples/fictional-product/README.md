# 虚构产品示例：TaskBoard

TaskBoard 是一个虚构的团队任务管理产品，只用于演示 FORGE Lite。

它不是任何真实产品，也不包含真实业务背景。

## 产品背景

- 产品名：TaskBoard
- 产品类型：团队任务管理工具
- 用户角色：工作区管理员、项目负责人、普通成员、访客
- 核心对象：工作区、项目、任务、评论
- 核心流程：创建项目、分配任务、更新任务状态、评论协作、查看进度

## 示例需求

```text
我们希望访客也可以在任务下发表评论，
但不希望他们成为完整的工作区成员。
请帮我判断这个需求的产品方向，以及是否需要进入 PRD。
```

## 应该怎么跑

这个需求不应该直接写 PRD。

建议流程：

1. 需求理解
2. 产品现状理解
3. 方向门禁
4. 方案设计
5. 方向确认后再生成 PRD

## 为什么需要方向门禁

这个需求触碰了：

- 角色模型
- 可见范围
- 权限边界
- 评论生命周期
- 通知行为

所以 Agent 应该先判断：

- 访客是什么角色？
- 访客能看到哪些任务？
- 访客能评论哪些内容？
- 评论是否会触发通知？
- 访客权限是否有过期机制？
- 访客评论是否和成员评论同等展示？

这些问题确认前，不应该直接进入 PRD 细节。
