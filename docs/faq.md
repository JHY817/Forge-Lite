# 常见问题

## FORGE Lite 严格意义上是 Agent 吗？

需要区分“仓库”和“运行中的系统”。

- FORGE Lite 仓库本身是 Agent 工作流、规则、状态、模板和 Eval 框架，不是独立模型或运行时。
- 当它由宿主 Agent 加载，并能读取产品配置、调用工具、维护阶段状态、根据 Rubric 反馈返工时，整个运行系统可以称为“产品经理工作流 Agent”或“Agent 系统”。

因此，对外更准确的说法是：“FORGE Lite 是一套用于构建产品经理工作流 Agent 的可配置框架。”

## FORGE Lite 是 PRD 模板吗？

不是。

FORGE Lite 是产品经理工作流 Agent 框架。PRD 生成只是其中一个模块。

## 它包含产品知识吗？

不包含。

使用者需要自己配置产品背景、知识库、代码库和业务 Rubric。

## 为什么刚安装时 Agent 不直接处理需求？

因为初始配置只包含占位符。Agent 如果继续生成方案，就会把通用常识冒充你的产品事实。配置门禁会先引导你完成产品背景和至少一个事实来源，然后才进入任务路由。

## 没有代码库也能用吗？

可以。

如果任务不依赖代码事实，可以跳过逆向工程模块。

## 非 SaaS 产品能用吗？

可以。

这套框架是通用的，但每个团队都需要配置自己的对象模型、流程、角色、Rubric 和模板。

## 每次都要跑完整流程吗？

不需要。

FORGE Lite 的原则是按需激活模块。简单任务可以只跑一个模块。

## 团队可以加自己的私有 Rubric 吗？

可以，而且应该加。

公开仓库只提供通用 Rubric。具体业务逻辑应该放在私有配置里。

## 为什么文件名是英文？

为了方便跨平台工具、链接和自动化脚本稳定识别。

当前正文中文优先，英文文档可在后续版本扩展。
