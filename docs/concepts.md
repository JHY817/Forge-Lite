# 核心概念

## 产品经理工作流 Agent

产品经理工作流 Agent 不是单纯写文档的工具。

它应该帮助产品任务在正确阶段推进：

- 需求理解
- 产品现状理解
- 事实补证
- 方案设计
- 原型准备
- PRD 生成
- 审计返工
- 经验沉淀

## 先判阶段，再做产物

很多产品任务失败，不是因为文档写得慢，而是因为太早进入产物。

FORGE Lite 的第一原则是：

```text
先判断任务处在哪个阶段，再决定产出什么。
```

比如：

- 需求不清楚，就先做需求理解。
- 现状不清楚，就先查产品现状。
- 方向不稳，就先做方向门禁。
- 方案已确认，再写 PRD。

## 方向门禁

方向门禁用于检查产品方向是否稳。

只要任务触碰这些内容，就要先审方向：

- 对象模型
- 归属边界
- 生命周期
- 权限
- 状态机
- 核心路径
- 系统隐式补偿

如果方向不稳，继续写 PRD 只会制造更多返工。

## 用户确认

FORGE Lite 默认人仍然负责关键决策。

常见确认点：

- 复杂任务执行前确认计划。
- 方案设计前确认关键事实。
- PRD 前确认方向。
- 原型执行前确认原型输入。
- 长期沉淀前确认是否写入经验库。

## 本地上下文

FORGE Lite 本身是通用框架。

真正让它有用的是你的本地上下文：

- 产品背景
- 知识库
- 代码库
- 业务规则
- 团队模板
- 私有经验沉淀

这些内容应该由使用者自己配置，不应该内置在开源仓库里。
