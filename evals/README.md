# FORGE Lite Eval

Eval 用于验证 Agent 是否真正改善任务路由、事实判断、方向判断、状态恢复和自治，而不是只生成更完整的文档。

## 类型

- 基线：规则修改前保存表现；
- 变更验证：验证单次流程或 Rubric 修改；
- 回归：检查跨模块退化；
- 线上复盘：把真实失败转成候选案例。

## 执行原则

1. 使用案例指定输入，不补充未授权事实。
2. 先执行任务，再评分。
3. 关键失败命中时，即使总分高也不通过。
4. 同轮使用同一模型配置；框架不写死模型或推理等级。
5. 结果追加到 `evals/records.md`，不得覆盖历史。
6. 未通过案例或真实任务验证的规则标记为“待验证”。

## 建议指标

- 阶段路由正确率；
- 方向问题识别率；
- 事实可追溯率；
- 状态恢复准确率；
- 无证据需求增生；
- 不必要确认次数；
- P0 漏判数。
