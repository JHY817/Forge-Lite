# 通用回归案例索引

这些案例不包含任何真实公司、产品、客户、代码库或内部文档。

| ID | 场景 | 主要验证 | 关键失败 |
|---|---|---|---|
| EVAL-001 | 用户直接要求新增按钮 | 问题收敛 | 未识别真实任务就实现初始方案 |
| EVAL-002 | 页面入口更少但后台关系复杂 | 方向门禁 | 用表层简化交换对象和生命周期复杂度 |
| EVAL-003 | 把已有能力扩展到新对象 | 状态机保真 | 未还原已有状态机就重新抽象 |
| EVAL-004 | 共享组件用于不同业务上下文 | 控件与策略边界 | 因复用组件就假设规则相同 |
| EVAL-005 | 多服务系统逆向 | 证据强度 | 单仓库没找到就断言产品没有能力 |
| EVAL-006 | 多版历史文档并存 | 状态恢复 | 跳过当前状态，采用失效草稿 |
| EVAL-007 | PRD 例外规则快速增长 | 回退 Loop | 继续补细则，不回退方向 |
| EVAL-008 | 简单规则变更 | 范围克制 | 为填模板生成无决策价值章节 |
| EVAL-009 | 原型新增方案外入口 | 原型输出验收 | 把视觉优化直接当成新需求 |
| EVAL-010 | 关键组织约束未知 | 信息对齐 | 能查却问用户，或高风险未知仍擅自决定 |
