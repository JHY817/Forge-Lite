# 通用回归案例索引

这些案例不包含任何真实公司、产品、客户、代码库或内部文档。

| ID | 场景 | 主要验证 | 关键失败 |
|---|---|---|---|
| EVAL-001 | 用户直接要求新增按钮 | 问题收敛 | 未识别真实任务就实现初始方案 |
| EVAL-002 | 页面入口更少但后台关系复杂 | 方向门禁 | 用表层简化交换对象和生命周期复杂度 |
| EVAL-003 | 把已有能力扩展到新对象 | 状态机保真 | 未还原已有状态机就重新抽象 |
| EVAL-004 | 共享组件用于不同业务上下文 | 控件与策略边界 | 因复用组件就假设规则相同 |
| EVAL-005 | 多服务系统逆向 | 证据强度 | 单仓库没找到就断言产品没有能力 |
| EVAL-006 | 多版历史文档并存 | 状态恢复 | 跳过当前状态，采用失效草稿 |
| EVAL-007 | PRD 例外规则快速增长 | 回退 Loop | 继续补细则，不回退方向 |
| EVAL-008 | 简单规则变更 | 范围克制 | 为填模板生成无决策价值章节 |
| EVAL-009 | 原型新增方案外入口 | 原型输出验收 | 把视觉优化直接当成新需求 |
| EVAL-010 | 关键组织约束未知 | 信息对齐 | 能查却问用户，或高风险未知仍擅自决定 |
| EVAL-011 | PRD 配图并写回在线文档 | 高保真工具路由、视觉质量与交付验证 | 高保真 Skill 已可用但未触发；只做到无重叠/无溢出；默认使用 Mermaid；写回后未检查实际预览和文档内容 |
| EVAL-012 | 首次安装后直接提出产品需求 | 配置门禁 | 把占位符当产品事实，或在产品背景和事实来源未配置时直接生成方案/PRD |

## EVAL-011 执行边界

- 输入：一份需写回在线文档的 PRD，同时包含对象关系、方案对比和少量顺序性说明。
- 期望：逐图判断是否有必要；先完成视觉简报；复杂正式配图实际调用 `fireworks-tech-graph` 或等价高保真 Skill 并生成 SVG/PNG；需要可编辑时再转换到平台画板；通过视觉 Rubric 后写回，并检查平台实际预览和文档内容。
- 禁止：为所有段落强行配图；高保真 Skill 已可用但未触发；统一使用 Mermaid；为画面丰富新增 PRD 未确认内容；把多个独立主题挤在一张图中；只检查语法、重叠或溢出，不检查 PNG 和平台实际预览。

## EVAL-012 执行边界

- 输入：全新安装，`config/` 仍为示例占位符；用户直接要求设计一个产品需求。
- 期望：Agent 先执行配置检查，进入配置引导，列出需先填写的产品背景和事实来源；不进入方案或 PRD。
- 配置后期望：配置门禁通过，Agent 输出阶段判断、模块激活、证据缺口和用户门禁，仍不应无条件直接写 PRD。
