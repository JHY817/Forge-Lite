# 问题严重度与风险接受

| 级别 | 定义 | 示例 | 是否允许带风险通过 |
|---|---|---|---|
| P0 | 导致方向错误、事实失真、越权、不可逆后果或核心交付不可用 | 对象模型错误、关键事实无证据、当前状态污染 | 不允许 |
| P1 | 明显影响主要场景、实现理解、历史兼容或造成较大返工 | 关键状态遗漏、主要入口规则冲突 | 仅用户明确接受后允许 |
| P2 | 不影响核心方向和主要交付 | 次要表达、低频提示、非关键结构问题 | Agent 可通知后推进 |

严重度按影响判断，不按修改工作量判断。

## 停止规则

- P0：停止下游推进，修复或回退根因阶段。
- P1：停在 approve 门禁前，由用户决定修复、缩小范围或接受风险。
- P2：记录到问题账本和交接摘要，可以继续。

## 复审范围

| 修改类型 | 复审方式 |
|---|---|
| 文字、排版、非语义结构 | 增量复审 |
| 局部规则，不改变对象和范围 | 复审问题项及相邻状态 |
| 范围、对象、归属、权限、生命周期、核心路径变化 | 下游全量重审 |
| 关键事实被推翻 | 回退根因阶段，下游审计失效 |

风险接受必须记录问题 ID、影响、原因、接受人和后续动作。
